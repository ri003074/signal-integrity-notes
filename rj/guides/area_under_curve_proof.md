# 「グラフの下の面積 = 積分」は本当か？ 実証

## 1. 具体例：速度グラフから位置を求める

### データ

時刻別の速度（秒ごと）:

```
t=0秒: v=0 m/s
t=1秒: v=10 m/s
t=2秒: v=10 m/s
t=3秒: v=5 m/s
t=4秒: v=0 m/s
```

---

## 2. グラフの下の面積を「長方形で計算」

速度グラフを描くと：

```
速度 (m/s)
    |
 10 |     ■ ■
    |     ■ ■ ■
  5 |         ■
    |         ■
  0 |_________■_________ 時刻 (秒)
    0 1 2 3 4
```

「グラフの下の面積」= 各時間区間での長方形の面積

### 計算：長方形の面積を足す

```
0～1秒: 幅1秒 × 高さ0 m/s = 0 m/s × 1秒 = 0 m
1～2秒: 幅1秒 × 高さ10 m/s = 10 m/s × 1秒 = 10 m
2～3秒: 幅1秒 × 高さ10 m/s = 10 m/s × 1秒 = 10 m
3～4秒: 幅1秒 × 高さ5 m/s = 5 m/s × 1秒 = 5 m

合計面積 = 0 + 10 + 10 + 5 = 25 m
```

**この 25 m が「速度グラフの下の面積」です。**

---

## 3. 積分で計算

積分の定義：

```
位置 = ∫ 速度 dt
```

コード的には（数値積分）：

```python
import numpy as np

# 時刻と速度
t = np.array([0, 1, 2, 3, 4])
v = np.array([0, 10, 10, 5, 0])

# 時間の刻み
dt = t[1] - t[0]  # = 1秒

# 積分：速度の面積を足す
position = np.sum(v[:-1] * dt)  # [0, 10, 10, 5] × 1 = 25 m
```

**結果 = 25 m**

---

## 4. 確認：2つの方法が同じ結果

| 方法 | 計算 | 結果 |
|------|------|------|
| グラフの長方形 | 0 + 10 + 10 + 5 | **25 m** |
| 積分（∫v dt） | np.sum(v × dt) | **25 m** |

**同じです！**

---

## 5. なぜ同じなのか？

積分の定義そのものが「グラフの下の面積」だからです。

数学的には：

```
∫[0→4] v(t) dt  = 「0秒から4秒までの速度グラフの下の面積」

この積分を「長方形に切る」と：

= (v[0] × dt) + (v[1] × dt) + (v[2] × dt) + (v[3] × dt)
= (0 × 1) + (10 × 1) + (10 × 1) + (5 × 1)
= 25 m
```

**つまり「グラフの下の面積を足す」=「積分を計算する」と同じ操作**

---

## 6. FailCountでも同じ

ヒストグラムをもう一度見ます：

```
bin1 (x=1): h=20個
bin2 (x=2): h=50個
bin3 (x=3): h=30個
```

ヒストグラムのグラフ（棒グラフ）：

```
個数
   |
50 |     ■
30 | ■   ■
20 | ■   ■
   |_■_■_■_________
    1 2 3  x
```

### 「右から足す」を長方形で見ると

各 x での FailCount（x以上の個数）：

```
x=1: 右側の棒を全部足す
    20個（x=1の棒）+ 50個（x=2の棒）+ 30個（x=3の棒）= 100個

x=2: 右側の棒を全部足す
    50個（x=2の棒）+ 30個（x=3の棒）= 80個

x=3: 右側の棒を全部足す
    30個（x=3の棒）= 30個
```

---

## 7. 「右から足す」も面積の言い方

ヒストグラムを「x方向」に見たとき：

```
FailCount(x) = 「x の右側にある棒の面積を全部足す」
            = 「ヒストグラムを右から積分」
```

数学的には：

```
F(x) = ∫[x→∞] h(t) dt
     = 「x から右側の h(t) を足す」
     = グラフの「右側面積」
```

---

## 8. 最終確認：図解

### 速度グラフ

```
v(t)
  |     面積
  |    ▲▲▲▲
10|  ▲▲▲▲▲▲▲▲
  |▲▲▲▲▲▲▲▲▲▲▲▲
  |_________________
  0  1  2  3  4  t

「グラフの下の面積」
= 0×1 + 10×1 + 10×1 + 5×1 = 25 m

積分 = ∫v dt = 25 m
```

### ヒストグラム（FailCount）

```
h(x)
  |
50|     ■
30| ■   ■    ← 「x=1以上」の面積
20| ■   ■       = 20+50+30 = 100
  |_■_■_■_
  1 2 3  x

「x から右側の面積」
= F(x=1) = 100

FailCount = ∫[1→∞] h(x) dx = 100
```

## 9. コード検証

```python
import numpy as np

# ===== 速度の例 =====
t = np.array([0, 1, 2, 3, 4])
v = np.array([0, 10, 10, 5, 0])
dt = 1

# グラフの下の面積（長方形で）
area_velocity_graph = np.sum(v[:-1] * dt)
print(f"速度グラフの下の面積: {area_velocity_graph} m")
# 出力: 25 m

# 積分
position = np.sum(v[:-1] * dt)
print(f"積分結果: {position} m")
# 出力: 25 m

print(f"同じか？ {area_velocity_graph == position}")
# 出力: True

# ===== ヒストグラムの例 =====
x = np.array([1, 2, 3])
h = np.array([20, 50, 30])

# 「右から足す」= FailCount
F = np.array([
    np.sum(h[0:]),   # x=1以上
    np.sum(h[1:]),   # x=2以上
    np.sum(h[2:])    # x=3以上
])
print(f"\nFailCount: {F}")
# 出力: [100 80 30]

# 「右側の面積」
right_side_area = np.sum(h[0:])
print(f"x=1以上の面積（右から足す）: {right_side_area} 個")
# 出力: 100 個

print(f"FailCount(x=1): {F[0]} 個")
# 出力: 100 個

print(f"同じか？ {right_side_area == F[0]}")
# 出力: True
```

---

## 10. 結論

**「グラフの下の面積 = 積分」は 100% 本当です。**

その理由：

1. 積分の定義そのものが「面積」です
2. グラフを長方形に切って足す = 積分を計算する
3. FailCountでも「右から足す」= 「右側の面積を足す」= 同じ操作

違いは：

- **普通の速度グラフ**：左から右へ足す `∫[0→t]`
- **FailCountのグラフ**：右から左へ足す `∫[x→∞]`

どちらも「グラフの面積を足す」ことに変わりはありません。
