# 一般的な微分積分 vs FailCount の比較

## 1. 一般的な微分積分（日常例）

### 例：車が走る距離・速度・加速度

時間 `t` に対して：

- **位置** `x(t)` = 時刻 `t` での車の位置  
  例: 10秒後に 100m の位置

- **速度** `v(t) = dx/dt` = 位置を時間で**微分**  
  意味：今この瞬間、1秒あたり何m進むか  
  例: 時速 36km = 10 m/秒

- **積分**（逆操作）：`x(t) = ∫v(t)dt`  
  意味：速度を足し上げると元の位置に戻る

---

### グラフで見ると

```
位置グラフ:     まっすぐ上昇
速度グラフ:     一定値（平坦）← 位置の傾き = 速度
```

**微分の意味：グラフの傾きを取ること**  
**積分の意味：グラフの下の面積を足すこと**

---

## 2. FailCount と ヒストグラムの関係

時間ではなく、**パラメータ** `x`（例：ジッタの大きさ）に対して：

- **ヒストグラム** `h(x)` = パラメータ `x` の値を持つデータが何個あるか  
  例: `x=0.5ns` の値を持つデータが 100個  
  → グラフ：棒グラフ（離散的）

- **FailCount** `F(x) = ∫[x→∞] h(t)dt` = `x以上` の個数を足し上げたもの  
  意味：`x` より右側にあるデータが全部で何個か  
  例: `x=0.5ns` 以上のデータが 500個

- **微分**（逆操作）：`h(x) = -dF/dx`  
  意味：FailCount の傾きを取るとヒストグラムに戻る

---

## 3. 対比表

| 項目 | 一般的な微分積分 | FailCount |
|------|-----------------|-----------|
| **基本の量** | 位置 `x(t)` | ヒストグラム `h(x)` |
| **微分した量** | 速度 `v(t) = dx/dt` | ？ |
| **積分した量** | 位置 `x = ∫v dt` | FailCount `F(x) = ∫h(u)du` |
| **変数** | 時間 `t` | パラメータ `x` |
| **グラフ** | 滑らかな曲線 | 階段状（後で滑らか化） |
| **物理的意味** | 1秒後の位置 → 今の速度 | `x以上`のデータ → ヒストグラムの高さ |

**ここが大事：**

|  | 一般的 | FailCount |
|---|---|---|
| 基本 | 位置 | **ヒストグラム** |
| ↓ 微分 | → 速度 | → **（ヒストグラムの傾き）** |
| ↓ 積分 | ← 位置 | ← **FailCount** |

---

## 4. 「右から足す」を微積で正当化

### 一般的な微分積分では：
```
速度v(t) を積分すると、位置x = ∫v dt
= 速度グラフの「下の面積」を足す
```

### FailCountでは：
```
ヒストグラムh(x) を積分すると、FailCount F = ∫[x→∞] h(t)dt
= ヒストグラムの「右の棒」を足す
= 「x以上」の個数
```

**なぜ「右」なのか？**  
→ FailCount は「x `以上`」と定義されているから。  
→ 数式では `∫[x→∞]` と書く（x から無限大まで）。  
→ グラフでは「x の右側」と見える。

---

## 5. 数値例：一般的な微分積分

### 速度から位置を求める（積分）

時刻別に速度がこうだったとします（秒ごと）:

```
t=0秒: v=0 m/s
t=1秒: v=10 m/s
t=2秒: v=10 m/s
t=3秒: v=0 m/s
```

積分（速度グラフの下の面積）:
```
x(0) = 0 m
x(1) = 0 + 0*1 = 0 m        （0秒から1秒間、速度0）
x(2) = 0 + 10*1 = 10 m      （1秒から2秒間、速度10）
x(3) = 10 + 10*1 = 20 m     （2秒から3秒間、速度10）
x(4) = 20 + 0*1 = 20 m      （3秒から4秒間、速度0）
```

微分（位置グラフの傾き）:
```
v(0~1) = (0-0)/1 = 0
v(1~2) = (10-0)/1 = 10     ← 元の速度に戻った
v(2~3) = (20-10)/1 = 10
v(3~4) = (20-20)/1 = 0
```

---

## 6. 数値例：FailCount

### ヒストグラムからFailCountを求める（積分）

ヒストグラムがこうだったとします：

```
bin1 (x=1): h=20個
bin2 (x=2): h=50個
bin3 (x=3): h=30個
```

**「右から足す」 = 積分**:
```
F(bin1) = 20 + 50 + 30 = 100個  （x=1以上）
F(bin2) = 50 + 30 = 80個        （x=2以上）
F(bin3) = 30個                  （x=3以上）
```

微分（FailCountの傾き = 隣同士の差）:
```
-(F(bin2) - F(bin1)) = -(80-100) = 20     ← 元のヒストグラムに戻った
-(F(bin3) - F(bin2)) = -(30-80) = 50
```

---

## 7. 最後の確認：何が違うのか？

### 共通点
- どちらも「微分と積分は逆操作」
- グラフの傾き ↔ グラフの下の面積

### 違う点

| | 一般的 | FailCount |
|---|---|---|
| 変数 | **時間** `t` | **パラメータ** `x` |
| 基本の量 | **位置** `x(t)` | **個数分布** `h(x)` |
| 積分方向 | 左から右へ `∫[0→t]` | **右から左へ** `∫[x→∞]` |
| グラフ形状 | なめらか | 階段状 |

**「右から」という理由：**  
FailCountの定義が「x `以上`」だから。  
一般的な積分は「0 から t まで」だけど、FailCountは「x から ∞ まで」だから、グラフでは「右」に見える。

---

## 8. 最後のポイント

**微分積分の本質は変わらない：**

- `微分` = グラフの傾きを見る  
- `積分` = グラフの面積を足す  

**FailCountと違うのは「何を積分するか」と「どの方向か」だけ**

- 一般的：速度を積分 → 位置に戻る  
- FailCount：ヒストグラムを積分 → FailCountになる（右から足す）  
- FailCount：FailCountを微分 → ヒストグラムに戻る

---

## 次のステップ

FailCountを積分したときの面積は、「ヒストグラムを戻す」のではなく、「FailCountをさらに累積した新しい量」です。

一般的な例なら：
- 位置 → 微分 → 速度
- 速度 → 積分 → 位置  
- 位置 → **積分** → **「移動距離の累積」**（別の量）

FailCountなら：
- ヒストグラム → 積分 → FailCount
- FailCount → 微分 → ヒストグラム  
- FailCount → **積分** → **「Fail Countの累積面積」**（別の量）
