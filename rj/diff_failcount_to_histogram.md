# Fail Count を微分するとヒストグラムになる理由

---

## X 軸の意味（重要）

2つのグラフは **同じ X 軸** を共有していますが、「X 軸が何を意味するか」が混乱しやすいポイントです。

### ② Fail Count グラフの X 軸

```
X 軸 = 閾値（しきいち）の位置
```

「X = 0.5 のときに何回失敗するか？」という **閾値を動かして測定** した結果です。

```
閾値 x を右へ動かすと →  判定が厳しくなる → 失敗しやすくなる → Fail Count が増える

        ジッタ分布
          ╭─╮
         ╭╯ ╰╮
        ╭╯   ╰╮
    ────┼──────────── 信号レベル
        │
       x=0.3 のとき: 分布の右半分（面積大）が閾値を超える → Fail Count 高
       x=0.7 のとき: 分布の右端だけ（面積小）が閾値を超える → Fail Count 低
```

> **Fail Count(x) = 「ジッタが閾値 x を超える確率」**

---

### ① Histogram グラフの X 軸

```
X 軸 = ジッタの値そのもの（Fail Count と同じスケール）
```

Fail Count を微分して得たヒストグラムの X 軸は「どの位置にジッタが存在するか」を示します。

> **Histogram(x) = 「ジッタが x 付近に存在する確率密度」**

---

### なぜ同じ X 軸で意味が変わるのか

「閾値 x を動かして失敗確率を計測する」という操作は、  
実は「ジッタがどの値に分布しているかを積分しながら調べている」のと同じことです。

```
                    ジッタ分布（PDF）
                       ╭─╮
                      ╭╯ ╰╮
    ─────────────────╭╯   ╰╮──────── ジッタの値（X軸）
                    ↑       ↑
                  x=0.3   x=0.7

    閾値を x=0.3 に設定 → ハッチ部分の面積 = Fail Count(0.3)
    ████████████████╭─╮
    ████████████████╯ ╰╮
    ───────────────╭╯   ╰╮───────

    閾値を x=0.7 に設定 → ハッチ部分の面積 = Fail Count(0.7)
                       ╭─╮
                      ╭╯ ╰█
    ─────────────────╭╯   ╰╮───────
```

閾値を少し右に動かしたとき（x から x+Δx に）、  
**面積がどれだけ減るか** が、まさに「その位置でのジッタの密度（PDF）」です。

```
Fail Count(x) − Fail Count(x+Δx)   ≈   PDF(x) × Δx
          （閾値を少し動かしたときの面積の変化）
```

これを Δx で割って極限を取ったのが微分であり、PDF になります。

---

### まとめ表

| | X 軸の意味 | Y 軸の意味 |
|--|-----------|-----------|
| ② Fail Count | 閾値の位置 | 閾値を超えたサンプルの割合（累積） |
| ① Histogram  | ジッタの値（= 閾値と同じスケール） | その位置でのジッタの密度（局所的） |

**2つのグラフの X 軸は物理的に同じスケールであり、  
「累積的に見るか（Fail Count）・局所的に見るか（Histogram）」の違いだけです。**

## 直感的なイメージ

まず数式なしで考えます。

Fail Count カーブは「急激に落ちているところ」と「緩やかなところ」があります。

```
Fail Count
100 │●●
    │   ●●
 50 │      ●●
    │        ●●
  0 │          ●●●
    └──────────────────── x
    ↑ ここは平坦        ↑ ここも平坦
    （ほとんど落ちない）   （ほとんど落ちない）
           ↑
       ここが急峻（最もよく落ちる）
```

**x = 0 付近（左端）に注目**してください。  
Fail Count はほぼ 100 のまま **平坦（ほとんど変化しない）** です。

これは何を意味するでしょうか？

> 「閾値を x=0 から少し右へ動かしても、Fail Count がほとんど減らない」  
> = **「その付近をジッタが通過することがほとんどない」**

つまり、左端（x=0）では **波形がほとんど通らない（ジッタが少ない）** ため、  
閾値を動かしても Fail Count が変わらず、ヒストグラムも低くなります。

逆に **中心付近（x ≈ 0.5）** では Fail Count が急激に落ちます。  
= **「その付近にジッタが集中している（波形がよく通る）」**  

```
Fail Count の傾き（左端）≈ 0   → ジッタがほぼない     → ヒストグラムが低い
Fail Count の傾き（中心）≈ 大  → ジッタが密集している  → ヒストグラムが高い
Fail Count の傾き（右端）≈ 0   → ジッタがほぼない     → ヒストグラムが低い
```

そのまま並べると、ヒストグラムが自然に浮かび上がります：

```
Fail Count の傾き（符号反転）＝ ヒストグラム

  x=0付近  中心付近  x=1付近
    低        高        低   ← ヒストグラムの高さ（= 正規分布の山型）
```

「急激に落ちているところ」は何を意味するでしょうか？

> **「その x の付近に、多くのジッタが集まっている（波形がよく通る）」** ということです。

つまり、Fail Count の **落ち方の急さ（= 傾き）** が  
**そこに存在するジッタの量（= ヒストグラムの高さ）** に対応しています。

```
Fail Count の傾きが急 → その x にジッタが密集 → ヒストグラムが高い
Fail Count の傾きが緩 → その x にジッタが少ない → ヒストグラムが低い
```

```
    │
```

```
  │  │  │

  │  │  │  │ │
  └──────────────── x
```

## 数式で確認

### ステップ① Fail Count の正体

Fail Count は「x より右側にジッタがある確率」= **補完 CDF（CCDF）** です。

```
Fail Count(x) = max_fail × P(ジッタ > x)
              = max_fail × (1 − CDF(x))
```

図で表すと：

```
CDF(x) = ∫_{-∞}^{x} PDF(t) dt

     PDF（山型）       CDF（S字上昇）    Fail Count（S字降下）
        │  ╭─╮           │       ╭──      1│────╮
        │ ╭╯ ╰╮          │    ╭──╯         │    ╰──╮
        │╭╯   ╰╮         │╭───╯           0│       ╰────
        └──────────       └──────────       └──────────
```

**Fail Count = 1 − CDF なので、CDF を上下に反転した形です。**

---

### ステップ② 微分すると何が出てくるか

微積分の基本定理より：

```
CDF(x) を微分すると PDF(x) になる

d/dx CDF(x) = PDF(x)
```

Fail Count は `1 − CDF(x)` なので：

```
d/dx Fail Count(x)  =  d/dx (1 − CDF(x))
                    =  0  −  PDF(x)
                    =  −PDF(x)
```

つまり：

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   −d/dx Fail Count(x)  =  PDF(x)  =  ヒストグラム      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

マイナス符号がつくのは「Fail Count は右に行くほど**減少**するカーブ」だからです。  
減少カーブの微分は負の値になるため、マイナスを掛けて正に戻しています。

## 全体の流れを図で整理

```
      PDF（ヒストグラム）              Fail Count（補完 CDF）
          │  ╭─╮                          1│────╮
          │ ╭╯ ╰╮                          │    ╰──╮
          │╭╯   ╰╮                        0│       ╰────
          └──────────                      └──────────

  ←─────── 積分 ──────────────────────────────────────────→
  ←─────── 微分（× −1） ──────────────────────────────────
```

| 方向 | 操作 | 変換 |
|------|------|------|
| 左 → 右 | 積分（+上下反転） | ヒストグラム（PDF） → Fail Count |
| 右 → 左 | 微分（× −1）      | Fail Count → ヒストグラム（PDF） |

---

## コードとの対応

```python
# ② Fail Count カーブ（補完 CDF）
failcount_float = 100.0 * 0.5 * erfc((x - center) / (sigma * sqrt(2)))
#                 ↑ これが max_fail × (1 − CDF(x)) に相当

# ① ヒストグラムを求める（Fail Count を微分して符号反転）
pdf_numerical = -np.gradient(failcount_float, x)
#               ↑ −d/dx を計算している
```

`np.gradient()` は数値微分（隣の点との差分から傾きを計算）です。  
実際の測定データからでも、同じ操作でジッタの分布形状を推定できます。

---

## まとめ

```
② Fail Count は「ジッタが x を超える確率」の累積カーブ（補完 CDF）

  → 急に落ちている部分   = そこにジッタが多い
  → 緩やかに落ちている部分 = そこにジッタが少ない

  → 「落ち方の急さ」= 傾き = 微分 = ヒストグラムの高さ

  したがって:  −d/dx (Fail Count) = ヒストグラム（PDF）
```
