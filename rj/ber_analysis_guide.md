# BER / Random Jitter 解析ガイド

`ber.py` を実行すると3段のグラフが表示されます。  
それぞれの意味と読み方を説明します。

---

## 前提知識：erfc（補完誤差関数）とは

グラフを理解するために、まず **erfc** を知っておく必要があります。

### 正規分布（ガウス分布）から始める

正規分布は「ノイズや誤差が中心付近に集まり、端に行くほど少ない」という  
自然界でよく現れる確率分布です。グラフにすると山型（ベル曲線）になります。

```
確率密度
  │     ╭───╮
  │   ╭─╯   ╰─╮
  │ ╭─╯         ╰─╮
  └──────────────────── x
         ↑
       中心（μ）
```

### CDF（累積分布関数）= 山を積分したもの

「x 以下になる確率」を表します。左から右に向かって 0 → 1 に増加する S 字カーブです。

```
CDF(x) = P(X ≤ x) = ∫_{-∞}^{x} 正規分布 dt
```

```
確率
1.0 │              ╭────
    │          ╭───╯
0.5 │       ╭──╯
    │   ╭───╯
0.0 └──────────────────── x
```

### 補完 CDF = erfc カーブ = 「x より大きくなる確率」

```
補完 CDF(x) = 1 − CDF(x) = P(X > x)
```

CDF を上下反転したものです。左から右に向かって **1 → 0 に減少**します。

```
確率
1.0 │────╮
    │    ╰───╮
0.5 │        ╰──╮
    │            ╰───╮
0.0 └──────────────────── x
```

**これが Fail Count カーブの正体です。**

> **注意：x の方向について**  
> x は「マージン（余裕）」を表すため、  
> - **x が小さい** = マージンが少ない = **失敗しやすい（Fail Count 大）**  
> - **x が大きい** = マージンが多い = **失敗しにくい（Fail Count 小）**  
>
> erfc カーブは左（x=小）が高く、右（x=大）が低くなります。  
> 「確率が上がる」のではなく、x が増えると Fail Count（失敗確率）は**下がります**。

### erfc の定義式

数学的には以下のように定義されています：

```
erfc(x) = 1 − erf(x)
        = (2/√π) × ∫_x^∞  e^(−t²) dt
```

- `erf` は「誤差関数（error function）」
- `erfc` はその補完（complementary）版 = `1 − erf`

コードでは `scipy.special.erfc` を使っています：

```python
from scipy.special import erfc

# Fail Count カーブを生成
base = 0.5 * erfc((x - center) / (sigma * sqrt(2)))
fail_count = base * 100  # 0〜100 にスケール
```

### 3つの関係を一言でまとめると

```
正規分布（山型）  ──積分──▶  CDF（S字上昇）  ──上下反転──▶  erfc（S字降下）
                                                                     ↑
                                                            Fail Count カーブの形
```

| 関数 | 形 | 意味 |
|------|----|------|
| 正規分布 PDF | 山型（ベル曲線） | ジッタの確率密度 |
| CDF | S字（左→右に上昇） | x 以下になる確率 |
| **erfc（補完 CDF）** | **逆S字（左→右に下降）** | **x より大きくなる確率 = Fail の確率** |

## ① Jitter Histogram（上段）

### 何を表しているか

**ジッタの確率分布（PDF）** を棒グラフで表示したものです。  
「各 x の位置でどれだけのジッタが発生しているか」の分布形状を示します。

### 縦軸・横軸

| 軸 | 内容 |
|----|------|
| 横軸 x | テスト条件（0.0〜1.0 に正規化） |
| 縦軸 | 確率密度（PDF の値） |

### ヒストグラムの作り方（Fail Count の微分）

Fail Count カーブ（補完 CDF）を x で微分すると、符号を反転した PDF になります：

```
PDF(x) = −d(FailCount)/dx
```

これは微積分の基本定理から導かれます：

```
CDF(x)       = ∫ PDF(t) dt     （PDF を積分すると CDF）
d/dx CDF(x)  = PDF(x)          （微分すると PDF に戻る）

FailCount ∝ 1 − CDF(x)        （補完 CDF）
→ −d/dx FailCount = PDF(x)
```

コードでは `np.gradient()` で数値微分しています：

```python
failcount_float = 100.0 * 0.5 * erfc((x - center) / (sigma * sqrt(2)))
pdf_numerical = -np.gradient(failcount_float, x)
```

### 3本の線・棒の意味

| 要素 | 色 | 意味 |
|------|----|------|
| 棒グラフ | 青（半透明） | Fail Count を数値微分して得たヒストグラム |
| 破線 | 灰 | 真値の正規分布 PDF（理論値） |
| 実線 | 赤 | 推定した σ_RJ と μ で描いた正規分布 PDF |

### 読み方のポイント

- 青棒と赤線が一致 → RJ 推定が正確
- ベル曲線（山型）になっている → ジッタが正規分布に従っている（純粋な RJ）
- 山型でない（二峰性など）→ DJ が混在している可能性
- 山の幅（σ_RJ）が狭い → ジッタが小さく、品質が良い信号

## ② Fail Count（中段）

![概念図](https://i.imgur.com/placeholder.png)

### 何を表しているか

「あるパラメータ x（タイミングマージンや電圧マージン）を変化させたときに、何回テストが失敗するか」を示すカーブです。  
①のヒストグラムを積分したものが、このカーブになります。

### 縦軸・横軸

| 軸 | 内容 |
|----|------|
| 横軸 x | テスト条件（0.0〜1.0 に正規化。実際は UI や mV など） |
| 縦軸 | Fail Count（0〜100。全テスト数に対する失敗回数） |

### 3本の線の意味

| 線 | 色 | 意味 |
|----|----|------|
| ● 点 | 青 | 実際の測定値（ノイズが乗っている） |
| 実線 | 赤 | `extract_rj()` で推定した erfc フィットカーブ |
| 破線 | 灰 | 理論上の真値（シミュレーションで与えたパラメータ） |

### カーブの形が erfc になる理由

x をマージン（余裕）として考えると、以下のように解釈できます：

```
x が小さい（マージンがない）→ ノイズが閾値を超えやすい → 失敗しやすい → Fail Count ≈ 100
x が大きい（マージンがある）→ ノイズが閾値を超えにくい → 失敗しにくい → Fail Count ≈ 0
```

グラフで確認すると：

```
Fail Count
100 │●●●
    │    ●●
 50 │       ●●
    │         ●●
  0 │           ●●●●
    └──────────────────── x
   小（マージンなし）  大（マージンあり）
```

**左側（x が小）= マージンが少ない = 失敗多い**  
**右側（x が大）= マージンが多い = 失敗少ない**

この確率は正規分布の補完 CDF（complementary CDF）で記述されます：

```
Fail Count(x) = max_fail × Q((x − μ) / σ_RJ)
              = max_fail × 0.5 × erfc((x − μ) / (σ_RJ × √2))
```

- **μ (mu)** : Fail Count が 50% になる x の位置（遷移の中心）
- **σ_RJ**   : Random Jitter の sigma。小さいほど急激に立ち下がる

### 読み方のポイント

- 赤と灰の破線が近ければ → RJ 推定が正確
- 青●が赤線に沿っていれば → ノイズが少なく測定品質が高い
- 赤線が急峻（σ_RJ が小さい）→ RJ が小さく、品質が良い信号

## ③ Q-scale plot（下段）

### 何を表しているか

Fail Count を **Q 関数の逆数スケール（Q-scale）** に変換してプロットしたグラフです。  
Random Jitter のみが存在する場合、このグラフは**必ず直線になる**という性質を利用します。

### 縦軸・横軸

| 軸 | 内容 |
|----|------|
| 横軸 x | テスト条件（Fail Count グラフと同じ） |
| 縦軸 Q⁻¹(BER) | BER を Q 関数の逆数で変換した値 |

### 変換の手順

```
① BER = Fail Count / max_fail       （0〜1 に正規化）
② Q⁻¹(BER) = −ndtri(BER)           （Q の逆関数を適用）
```

`ndtri` は標準正規分布の逆累積分布関数（逆 CDF）です。

### なぜ直線になるのか

BER の式を変形します：

```
BER(x) = Q((x − μ) / σ_RJ)

両辺に Q⁻¹ を適用：
Q⁻¹(BER) = (x − μ) / σ_RJ
          = (1/σ_RJ) × x  −  μ/σ_RJ
```

これは `y = slope × x + intercept` の**直線の式**です。

```
slope     = 1 / σ_RJ   → σ_RJ = 1 / slope
intercept = −μ / σ_RJ  → μ = −intercept / slope
```

### 2本の線の意味

| 線 | 色 | 意味 |
|----|----|------|
| ● 点 | 青 | 実測 BER を Q⁻¹ 変換した値 |
| 実線 | 赤 | 直線フィット結果。傾き = 1/σ_RJ |

### 読み方のポイント

- 青●が赤線（直線）に乗っている → 純粋な **Random Jitter（Gaussian）** のみ
- 青●が直線からずれている → **Deterministic Jitter（DJ）** が混在している可能性
- 直線の傾きが急 → σ_RJ が小さい（少ないジッタ）
- R² が 1 に近い → フィットの精度が高い

### Q⁻¹(BER) = 0 の意味

縦軸が 0 の水平線（BER=50% line）と直線が交わる x の値が **μ**（遷移中心）です。

## 3つのグラフの関係まとめ

```
① ヒストグラム（PDF）        ← ジッタの形を直接見る
      ↓ 積分
② Fail Count（補完 CDF）     ← 測定データの形
      ↓ Q⁻¹ 変換
③ Q-scale plot（直線）       ← σ_RJ を数値として取り出す
      ↓ 直線の傾きの逆数
  σ_RJ = Random Jitter
```

| グラフ | 数学的な意味 | 実用的な用途 |
|--------|-------------|-------------|
| ① Histogram | PDF | ジッタ分布の形状確認・DJ の検出 |
| ② Fail Count | 補完 CDF | BER カーブの確認 |
| ③ Q-scale | 直線フィット | σ_RJ と μ の数値抽出 |

---

## パラメータの意味

| パラメータ | 説明 | 影響 |
|-----------|------|------|
| `sigma` (σ_RJ) | Random Jitter の標準偏差 | 小さいほど急峻なカーブ |
| `center` (μ) | Fail Count が 50% になる x の位置 | カーブの左右シフト |
| `noise_sigma` | 測定ノイズの大きさ | 0 にするとノイズなしの理想カーブ |
| `ber_range` | フィットに使う BER の範囲 | 端（0 や 1 に近い領域）は Q⁻¹ が発散するため除外 |
